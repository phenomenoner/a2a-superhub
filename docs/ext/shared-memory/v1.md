# A2A Superhub shared-memory extension v1

Canonical URI:
`https://phenomenoner.github.io/a2a-superhub/ext/shared-memory/v1`

Status: public extension contract; the legacy v1 coordination runtime does not
advertise an A2A 1.0 implementation of this extension.

An A2A 1.0 Agent Card may list this extension only when the same endpoint also
publishes a capability manifest containing `apiVersions: ["memory.v1"]`. A
client using the extension includes the canonical URI in the A2A message
`extensions` list and sends memory assertions as ordinary untrusted data parts.

The extension does not alter A2A authentication or authorize memory access.
The Superhub principal and scope registry remains authoritative. A request body
cannot select an author, expand visibility, acknowledge another consumer, or
turn a tool annotation into permission.

## Data shape

The JSON `data` Part carries one of these operation hints:

- `memory.note.create`: immutable additive assertion; requires an idempotency key.
- `memory.note.reference`: a `mem_<uuid>` provenance link.
- `memory.inbox.reference`: a consumer-bound opaque cursor reference.

The operation hint selects a supported workflow; the canonical HTTP schemas in
`schemas/` still validate the actual request and response. Unsupported extension
use fails before dispatch and does not silently fall back to legacy payloads.
