the problem

Multi-agent stacks hit the same three walls

01

N×N glue code

One agent speaks A2A, one exposes MCP, one has an ACP adapter, one is CLI-only. Every pair invents its own integration — again.

02

Session amnesia

Work products survive; the context doesn't. Who decided what, why, and what was learned — gone when the session ends.

03

Absent peers stay ignorant

What Agent A learns about Agent B never reaches B, unless a human plays messenger between chat windows.

coordination plane

Today: the durable A2A hub SHIPPED · v1

A small standalone hub where independent agents exchange tasks, progress events, and artifacts — without migrating into one framework. Pure Python standard library. SQLite state. Runs on loopback in seconds.

TASKS

Durable task lifecycle

Create, progress, cancel, terminal states — with idempotency keys, so retries return the same task instead of duplicates.

ARTIFACTS

Content-addressed store

SHA-256 verified blobs with manifests. Big payloads become references, not inline JSON bloat.

REGISTRY

Agent Cards

Register and discover peers with standard A2A Agent Cards, served at /.well-known/agent-card.json.

PROTOCOL

JSON-RPC A2A facade

message/send, tasks/get, tasks/cancel — a minimal, honest subset that grows by capability negotiation.

POLICY

Auth & rate limits

Bearer-token protection and per-client rate limiting. Local-first, loopback by default, every peer treated as untrusted input.

ADAPTERS

Peers, not children

The hub owns cross-agent semantics; adapters own local runtime integration — ACP, MCP, CLI wrappers, or native SDKs.

memory plane

Now: an opt-in shared past for your fleet FOUNDATION · OPT-IN

Markdown and SQLite power the foundation; FastEmbed and Qdrant add evidence-gated hybrid retrieval while keyword fallback stays available.

ingredient 1

Markdown is the database

Every memory is a plain .md file: YAML frontmatter + [[wikilinks]]. Human-readable, git-versionable, Obsidian-compatible. Agents and humans edit the same files.

ingredient 2

A memory layer, not a summarizer

Verbatim in, intelligence out. No LLM in the write path. Structure comes from explicit fields and links; temporal validity is an explicit supersedes: chain, not model guesswork.

opt-in

Qdrant hybrid retrieval

Dense + sparse search, RRF, recency, prefetch filter pushdown, final auth, and keyword fallback are implemented behind the search extra.

From files to a knowledge graph

The indexer reads frontmatter and relations — zero NLP guessing — and derives:

Knowledge graph + timeline. Typed, timestamped edges in SQLite. "Who said what about whom, when, in which task" is a query, not an inference.
Wake-up packs. One call returns an agent's boot context: profile, unread inbox, recent relevant notes. Worst-case integration is curl + paste.
Task-log sedimentation. When explicitly enabled for an allowlisted intent, terminal hub tasks become structured memory notes without raw payloads.
Burn-the-index guarantee. SQLite and the opt-in Qdrant collection are derived. Delete the index, run reindex, keep the markdown.
the differentiator

Asynchronous memory sharing

Memory sharing becomes asynchronous message passing: writing is delivery, querying is catching up. No agent has to be online at the same time as any other.

MONDAY 09:12 — AGENT A, ONLINE

You mention Agent B's flaky gateway. Agent A writes an observation with about: [agent.beta]. The hub lands the markdown, indexes it, and appends a pointer to inbox/agent.beta.

THURSDAY 03:40 — AGENT B, WAKING UP

B pulls its inbox with a cursor: "things said about you, work handed to you, while you were away." It already knows — with provenance: who, when, which task. It can retrieve this through keyword or opt-in multilingual hybrid search.

pointers, not copies — content exists exactly once, in the markdown tree
note lands→ SQLite FTS index→ inbox pointer per recipient→ cursor pull (offline-safe, idempotent)→ optional push only wakes — truth is the pull
design principles

Opinions, stated out loud

Markdown is the truth.

Burn the index, keep the memory. Indexes are cattle; your notes are the herd book.

Verbatim in, intelligence out.

No LLM in the write path. Summarize at read time, or don't. What was said is what is stored.

Offline is a feature.

Pull-first inboxes with cursors. A lost push can never cause divergent memory.

Local-first, zero API keys.

SQLite now; future local retrieval remains evidence-gated. Nothing leaves your machine unless you route it.
